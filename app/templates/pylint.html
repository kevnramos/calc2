Tips for beginners

The importance of Testing

Testing your code is one of the foundations of good programming

This can easily be done using a package in python called Pytest. First we will check if pytest is installed by running:

pytest --version

If the above does not give an output like pytest 6.2.5 then you will have to install pytest by running the following command:

pip install pytest

Linting with pytest

A linter is "a static code analysis tool used to flag programming errors, bugs, stylistic errors and suspicious constructs."

It helps you keep your code up to standard, and keeps the code throughout your codebase looking consistent

Luckily there is a linter included with pytest, which we just installed.

To lint your program, simply run:

pytest --pylint

Testing your code coverage

The tests you write should cover all the statements and methods, you've written throughout your program

This is called hitting all of your 'branches'

To test your code coverage, simply run:

pytest --cov

You can also easily lint and test your code coverage at the same time by simply running:

pytest --pylint --cov

Glossary of OOP Terms


Factory: A factory, as the name suggests, is an object that builds other objects.

Instantiation: Instantiation is when we create an object ourselves. For example a class Parrot, is instantiated with blu = Parrot("Blu", 10)

Class: A Class is code specifying a particular object in a program or application.

Object: An object can be many things in programming. It can be a variable, an array, a function, a method, etc. A class is even an example of an object.

Namespace: A namespace is a scope that identifies types, functions, variables, etc. This helps organize code and prevent name collisions.

Constructor: It is a subroutine of a class used to create an object of that class.

Fixture: A fixture is something regarding testing and predefines values used to run tests.

Type Hint: Type hinting shows you the type of value in your python code which lets you know what you are working with so you know how to manipulate it if needed.

Type Cast: This allows for the conversion of types in python.

Unit Test: Unit testing is testing your code in small units.

Static: Being static means method or the variable is the same regardless of the instance of the object. Meaning the class does not have to be instantiated.

Class Method: A class method is a method that is tied to the class itself and not an object of the class.

Design Patterns Used: In object-oriented programming there are many design patterns. Facade, which hides a program's complexity. Encapsulation, which only gives access to certain methods and variables.

Facade: A facade in object-oriented programming is a design patttern used to hide the complexities behind a program. Similar to what a facade is in architecture, it provides a pretty and simple interface for the user.

Method: Methods are functions defined inside of a class. They can define properties of an object

Property: Properties are characteristics of an object or class.

Encapsulation: Encapsulation allows us to hide and show specific methods and variables we want. We don't always want our methods or variables to be exposed to the user and easily modifiable.
