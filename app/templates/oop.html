Encapsulation

Encapsulation is the idea of grouping data and methods that work together within one unit.

To change the value of an object within a class, you have to do it using that specific classes method

A class itself is an example of encapsulation. A class holds different methods and variables withing itself, demonstrating encapsulation

Inheritance

Inheritance, in object-oriented programming, inheritance is the proccess of basing an object or class upon another object or class.

For example a rectangle class can inherit the square class, because a rectangle is a class, but a square is not a rectangle

Inheritance helps reduce redundancy throughout our calculator program. Instead of solely basing each attribute of the calculator in separate places, we use inheritance to basically build the calculator in one place, calculator.py

Polymorphism

In the simplest way possible, polymorphism is basically an overloaded function.

As you can see in the example code below from geeksforgeeks.org
The function Multiply() in the class MultiplyFun, can be called with either 2 or 3 parameters.

The end result will be the same, because regardless of how many parameters you pass to the function,
you will get the product of your parameters.

  // Java program for Method overloading

class MultiplyFun {
    // Method with 2 parameters

    static int Multiply(int a, int b)
    {
        return a * b;
    }

    // Method with the same name but 3 parameters

    static int Multiply(int a, int b, int c)
    {
        return a * b * c;
    }

Python has some built-in polymorphic functions like the print() function.

Regardless of what you pass through the print() function, you will get a result on the terminal.
You can pass a list, an integer, float, tuple, string, etc, and the print() function will always work
in the same way. This is an example of polymorphism.


Abstraction

Abstraction allows the programmer to expose certain methods or variables. This makes it easier for the end user, or for other programmers to understand your code.

This alone is one of the most important concepts of object-oriented programming

This reduces complexity for the end user, and also makes reading code much easier

Making code easy to read is the epitome of what programming should be because you will always be collaborating with other people on big projects in the real world.
